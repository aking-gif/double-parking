🛠️ أداة تشخيص وإصلاح بيانات KV

تعمل بدون تسجيل دخول. تستدعي الـ Worker مباشرة لتحديد الملف التالف وإصلاحه.

① فحص الحالة لم يبدأ

يفحص: هل الـ Worker شغّال؟ هل تسجيل الدخول يفشل بسبب JSON تالف؟ في أي موضع؟

② فحص جميع مفاتيح KV الحرجة لم يبدأ

يحدد المفتاح التالف بالضبط (users_v1 / sops_v1 / deps_v1 / activity_v1 / custom_depts_v1).

③ استعراض المفتاح التالف لم يبدأ

يجلب القيمة الخام ويعرض السياق حول الموضع التالف لتعرف ما الذي يجب إصلاحه.

④ إصلاح تلقائي لم يبدأ

يحاول تصليح الـ JSON تلقائياً (إغلاق أقواس، حذف فواصل ضائعة، قص الجزء التالف). يعمل نسخة احتياطية أولاً باسم users_v1__backup_BEFORE_FIX.

⑤ إصلاح يدوي (الأكثر أماناً) لم يبدأ

انسخ القيمة من خطوة ③، أصلحها في jsonlint.com، الصق هنا، احفظ.

⚠️ ملاحظة هامة

هذه الأداة تستخدم endpoints /api/debug/* في الـ Worker. إن لم تكن موجودة، أضف الكود التالي في worker.js قبل return json({error:"not-found"...}, 404, req); النهائي، ثم ادفع للـ Cloudflare. تجد الكود في أسفل الصفحة.
